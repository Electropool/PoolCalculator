 1.3 SPA CRAWLABILITY (BIGGEST TECHNICAL SEO ISSUE)
-------------------------------------------------------------------------------
STATUS: FAIL — HIGH IMPACT
SEVERITY: CRITICAL

The site is a pure CSR (Client-Side Rendering) React SPA. When Googlebot or
any crawler requests any URL, the server sends back a nearly empty HTML page:

  <!DOCTYPE html>
  <html lang="en">
  <head>
    <title>PoolCalculator – Fast & Simple Electronics Calculators</title>
    ...
  </head>
  <body>
    <noscript>You need to enable JavaScript to run this app.</noscript>
    <div id="root"></div>
  </body>
  </html>

This is what the raw HTML response looks like BEFORE JavaScript executes.

Googlebot CAN execute JavaScript ("second wave indexing"), but:
  1. It processes JS-rendered pages slower than static HTML
  2. New/low-authority domains have limited crawl budget
  3. Dynamic content loaded after user interaction (calc results) is NEVER
     indexed
  4. The title in index.html is the same for ALL pages until JS runs
  5. Any crawl error or JavaScript exception = empty indexed page

CURRENT PAGE METADATA STATE IN index.html:
  Title:       "PoolCalculator – Fast & Simple Electronics Calculators" (same for all routes)
  Description: "Free online electronics calculators for students and engineers."

This means if Googlebot crawls /led-calculator without executing JS, it sees
the homepage title and description. This creates duplicate metadata across all
routes from the crawler's perspective.

SOLUTION: Implement Static Prerendering (react-snap)

  STEP 1: Install react-snap
    npm install --save-dev react-snap

  STEP 2: Add postbuild script to package.json:
    "scripts": {
      "build": "react-scripts build",
      "postbuild": "react-snap"
    }

  STEP 3: Add react-snap config to package.json:
    "reactSnap": {
      "source": "build",
      "destination": "build",
      "include": [
        "/",
        "/led-calculator",
        "/resistor-color-code",
        "/smd-code",
        "/capacitor-code",
        "/inductor-code",
        "/about",
        "/contact",
        "/privacy-policy",
        "/terms"
      ],
      "puppeteerArgs": ["--no-sandbox", "--disable-setuid-sandbox"]
    }

  STEP 4: Update src/index.js to use hydrate instead of render:
    import { hydrateRoot, createRoot } from 'react-dom/client';
    const rootElement = document.getElementById('root');
    if (rootElement.hasChildNodes()) {
      hydrateRoot(rootElement, <React.StrictMode><App /></React.StrictMode>);
    } else {
      createRoot(rootElement).render(<React.StrictMode><App /></React.StrictMode>);
    }

  RESULT: After running npm run build, each route will have its own
  pre-rendered HTML file with correct title, description, canonical, and
  visible content — fully readable by Googlebot without JS execution.

  ALTERNATIVE (if react-snap causes issues):
    Add a server-side prerender step using Puppeteer, or consider migrating
    the project to Next.js in the future for proper SSG/ISR support.

IMPACT OF FIXING: This single change could improve indexing completeness
from ~40% to ~95% of pages and dramatically improve SERP appearance timelines.

      return (

          {items.map((item, i) => (
            <span key={item.path}>
